## 1. 产品简介

GoalOS 是面向人类目标的个人操作系统。你只需要说出"我要什么"——系统负责安全地实现它。

### 核心链路

```
你表述目标 → 系统生成任务 → 安全执行 → 实时监控 → 交付结果
```

### 不是

- 不是聊天机器人——你不是来闲聊的
- 不是 Workflow 引擎——你不需要画流程图
- 不是编程工具——你不需要写代码

### v1.1.0 新能力

- **CompletionContract**: 系统和你确认"什么叫完成"——不再盲目执行。每次执行前你会看到成功标准和验收条件
- **Flow 执行模板**: 同一类目标每次按相同标准流程执行——结果可预测
- **Multi-LLM 交叉验证**: AI 产出的代码经过多个模型独立审查——更可信
- **自修正**: 执行失败时系统自动分析原因并修正——不需要你盯着
- **运行可观测**: Timeline 中查看每个Action的执行状态、分歧详情、修正记录

## 2. 部署文档

### 2.1 系统要求

| 项目 | 最低要求 |
|------|---------|
| 操作系统 | macOS 12+ / Linux (kernel 5.x+) |
| 内存 | 8 GB RAM |
| 磁盘 | 500 MB 可用空间 |
| 网络 | 需要 LLM API 访问（Anthropic / OpenAI / Ollama） |
| 其他 | Chrome/Chromium（浏览器自动化需要） |

### 2.2 安装

```bash
# 1. 下载二进制
# 从 releases 页面下载 goalos-daemon 和 goalos

# 2. 赋予执行权限
chmod +x goalos-daemon goalos

# 3. 移动到 PATH
mv goalos-daemon /usr/local/bin/
mv goalos /usr/local/bin/

# 4. 验证安装
goalos version
```

### 2.3 配置

```bash
# 设置 LLM API Key（二选一）
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENAI_API_KEY="sk-..."

# 可选：设置本地 Ollama
export GOALOS_LLM_PROVIDER="ollama"
export GOALOS_LLM_MODEL="llama3"

# 配置文件（可选。默认值已可用）
# ~/.goalos/config/daemon.yaml
```

### 2.4 启动

```bash
# 启动 Daemon
goalos-daemon &

# 验证运行
goalos health
# 输出: GoalOS Daemon: 运行中
```

### 2.5 卸载

```bash
# 停止 Daemon
goalos daemon stop

# 删除二进制
rm /usr/local/bin/goalos-daemon /usr/local/bin/goalos

# 删除数据（可选）
rm -rf ~/.goalos/
# 注意：~/Goals/ 目录中的产出物不受影响——它们是你的文件
```

## 4. 功能说明

### 4.1 Goal 创建

**功能**：用自然语言描述目标。系统自动拆解为可执行的任务图。

**示例**：
- "开发一个面向中小企业的 CRM 系统"
- "分析欧洲市场需求并生成报告"
- "每天维护公司 TikTok 账号"

**模糊目标处理**：如果目标描述不够清晰，系统会反问澄清。你不说清楚——系统不会猜。

### 4.2 任务执行

**流程**：Goal → MissionGraph → Action 执行 → 结果交付。

系统支持的操作类型：
- 文件读写
- Shell 命令执行
- 浏览器自动化
- API 调用
- 代码生成

每个 Action 在独立隔离沙箱中执行。沙箱间互不影响。

### 4.3 安全治理

**所有执行必经安全检查。** 五层检查：

| 层级 | 说明 |
|------|------|
| Policy | 规则匹配（如"禁止删除生产文件"） |
| Capability | 权限校验（如"这个操作需要 shell.execute 能力"） |
| Risk | 风险评估（L0 无风险 ~ L5 关键风险） |
| Approval | L3+ 需人工审批 |
| Audit | 全量审计。不可关闭 |

**自治级别**：

| 级别 | 说明 |
|------|------|
| observe | 仅观察。不执行任何操作 |
| suggest | 建议模式。执行前提醒 |
| approve | 审批模式。L3+ 需人工批准（默认） |
| autonomous | 全自动。仅在严重异常时暂停 |

```bash
goalos config set autonomy-level approve
```

### 4.4 Persona（系统声音）

系统默认使用**简洁**风格。你可以切换：

```bash
goalos config set persona concise   # 简洁直接（默认）
goalos config set persona warm      # 温暖友好
goalos config set persona minimal   # 极简纯文本
```

### 4.5 CLI 命令参考

```bash
# 创建并执行目标
goalos "<目标描述>"
goalos new "<描述>" --async

# 查看与管理
goalos list [--status running]
goalos status <id> [--json]
goalos pause / resume / stop <id>

# 审批
goalos approve / reject <id>

# 日志与审计
goalos log <id> [--format jsonl]
goalos events export <id>

# 配置
goalos config set autonomy-level approve
goalos config set persona concise
goalos config get <key>

# Daemon 管理
goalos daemon status / stop / restart / logs

# Skill 管理
goalos skill search / install / list / info

# 系统
goalos health / version / help
```

### 4.6 Web UI

打开 `http://localhost:18920/`：

- **目标输入**：顶部输入框。输入目标→回车或点击"开始执行"
- **目标列表**：实时显示所有 Goal 的状态和进度
- **Watcher 弹窗**：高风险操作自动弹出审批对话框

## 5. 错误说明

### 5.1 启动错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `PID lock failed` | Daemon 已在运行 | `goalos daemon stop` 停止旧进程 |
| `cannot find home dir` | 无法获取用户目录 | 检查 `$HOME` 环境变量 |
| `port 18920 already in use` | 端口被占用 | 设置 `GOALOS_PORT=18921` |

### 5.2 执行错误

| 错误码 | 说明 | 处理 |
|--------|------|------|
| `GOAL_NOT_FOUND` | 目标不存在 | 检查 goal_id 是否正确 |
| `INVALID_REQUEST` | 请求参数无效 | 检查输入格式 |
| `GOAL_NOT_RUNNING` | 目标不在执行状态 | 只有 Running 状态的 Goal 可暂停/恢复 |
| `DAEMON_NOT_READY` | Daemon 未就绪 | 等待几秒后重试 |
| `INTERNAL_ERROR` | 系统内部错误 | 查看 `~/.goalos/logs/daemon.log` |
| `UNAUTHORIZED` | 未授权 | 检查 Token 配置 |
| `TOKEN_EXPIRED` | Token 过期 | 重新获取 Token |

### 5.3 LLM 错误

| 现象 | 原因 | 解决 |
|------|------|------|
| Goal 创建后一直停留在 Draft | LLM API 不可用 | 检查网络 + API Key |
| MissionGraph 解析失败 | LLM 返回格式异常 | 系统自动 fallback 到 3 节点默认图 |
| Token 预算耗尽 | Goal 消耗超过 1M token | 在 Goal 页面追加预算 |

### 5.4 存储错误

| 现象 | 原因 | 解决 |
|------|------|------|
| `STORAGE_ERROR` | 磁盘满或权限不足 | 检查 `~/.goalos/` 目录权限和磁盘空间 |
| events.jsonl 损坏 | 异常关机导致 | Daemon 启动时自动跳过损坏行 |

### 5.5 数据恢复

GoalOS 所有数据存储在：

```
~/Goals/          # 你的产出物（文档、代码、报告）
~/.goalos/        # 系统状态（可重建）
```

- **备份**：直接复制 `~/Goals/` 即可
- **恢复**：重新安装 GoalOS 后，将备份的 `~/Goals/` 放回原位
- **崩溃恢复**：Daemon 重启后自动从 events.jsonl 恢复状态

## 6. 常见问题

**Q: GoalOS 需要联网吗？**

部分需要。LLM 规划需要 LLM API。本地文件操作和 Shell 命令在离线时正常工作。

**Q: 我的数据在哪里？**

`~/Goals/<目标名>/` ——普通文件。你可以用 Finder、VS Code、Git 操作它们。GoalOS 不锁定你的数据。

**Q: 如何卸载？**

删除 `goalos-daemon` 和 `goalos` 二进制即可。`~/Goals/` 中的文件不受影响——它们是你的。

**Q: 支持哪些 LLM？**

Anthropic (Claude)、OpenAI (GPT-4o)、Google (Gemini)、本地 Ollama。通过配置切换。

**Q: 如何贡献 Skill？**

Skill 遵循 agentskills.io 开放标准。编写 Markdown + YAML Frontmatter → 发布到 ClawHub → 其他用户可通过 `goalos skill install` 安装。
